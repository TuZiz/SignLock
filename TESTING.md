# SignLock 手测矩阵

这份文档用于发布前手工验收。建议至少准备 2 个普通玩家账号和 1 个管理员账号，并在 Spigot、Paper 或 Folia 环境中各完成一轮关键流程。

## 测试环境

- 服务端：Spigot、Paper、Folia
- Java：与当前构建环境一致
- 命令别名：`/signlock`、`/bl`
- 主要方块：单箱、双箱、木桶、门、熔炉、潜影盒

## 核心流程

1. 玩家手持普通牌子右键单箱，确认会自动创建锁牌并消耗 1 个牌子。
2. 玩家手持普通牌子右键双箱任意半边，确认双箱整体按一把锁保护。
3. 玩家对已上锁容器使用 `Shift + 右键`，确认不会误创建新锁。
4. 锁主人普通右键自己的锁牌，确认会进入 GUI 管理界面。
5. 锁主人潜行右键自己的锁牌，确认仍走原生牌子编辑回退。
6. 非主人尝试编辑锁牌内容，确认会被拦截并收到提示。

## 命令与授权

1. 对准有效锁牌执行 `/bl add <玩家名>`，确认玩家被加入授权列表。
2. 主锁牌写满后继续执行 `/bl add <玩家名>`，确认会自动生成 `[更多用户]` 扩展牌。
3. 对准锁牌执行 `/bl remove <玩家名>`，确认目标玩家被移除。
4. 移除最后一块扩展牌中的最后一名玩家后，确认空扩展牌会被清理。
5. 对准锁牌执行 `/bl info`，确认会显示所有者、当前权限、目标摘要、授权信息和扩展牌数量。
6. 执行 `/signlock reload`，确认配置重载成功并返回清晰提示。

## 保护矩阵

1. 未授权玩家尝试打开单箱、双箱、门、熔炉，确认会被拦截。
2. 未授权玩家尝试破坏受保护方块本体，确认会被拦截。
3. 未授权玩家尝试破坏主锁牌或扩展牌，确认会被拦截。
4. 漏斗、发射器等自动化尝试输入或抽取已上锁容器，确认会被拦截。
5. 活塞推动锁牌、受保护方块，或推动普通方块撞向受保护结构，确认会被拦截。
6. 水流或爆炸影响锁牌和受保护方块，确认会被拦截。

## 身份与兼容

1. 被授权玩家改名后重新进服，确认旧锁仍可正常访问。
2. 管理员在开启 `admin-bypass` 时验证可绕过管理限制。
3. Folia 环境下多玩家同时加锁、授权、开箱，确认没有报错。

## Phase 6 Smoke Checklist

1. 锁主人普通右键锁牌时，GUI 中能看到“可管理”权限标签，并保留添加、移除、选择等管理控件。
2. 已授权但非主人的玩家普通右键同一块锁牌时，能打开只读摘要 GUI，且不会出现添加、移除或选择控件。
3. 未授权玩家普通右键锁牌时，不会打开完整摘要 GUI，也不会看到完整授权名单。
4. `/bl info` 对主锁牌和 `[更多用户]` 扩展牌都输出同一套摘要语言，双箱目标描述必须稳定显示为同一个共享锁目标。
5. 在 Paper 与 Folia 上分别验证一次：GUI 点击、聊天输入授权、`/bl info`、双箱摘要显示与容器保护都没有线程或调度错误。

## 发布前结论

- 所有核心流程、保护矩阵和 Phase 6 smoke checklist 都通过：可以发布。
- 任一条失败：先修复，再重跑受影响条目。
