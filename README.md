#
插件介绍

SignLock` 是一个基于牌子语义的容器与交互方块保护插件，目标是让服主和玩家都能用最直观的方式完成“上锁、授权、查看状态、管理名单”。点：

- 上锁逻辑稳定，不因为单箱、双箱或入口不同而出现保护漂移。
- 默认中文配置可直接上线，不需要先大改文案。
- 玩家依旧能按熟悉的“牌子锁”方式使用，同时获得更现代的 GUI 管理体验。
- 在 Spigot / Paper / Folia 环境下尽量保持一致行为。
- 直接手持牌子右键可上锁，不强制要求先手写特殊格式。
- 单箱、双箱、木桶、熔炉、漏斗、潜影盒、门、活板门、栅栏门等目标统一纳入可保护范围。
- 双箱按共享目标处理，避免点击左右半边时出现权限不一致。
- 锁主人普通右键锁牌可直接打开管理 GUI。
- 潜行右键仍保留原版牌子编辑回退，不强行打断老习惯。
- 支持通过 GUI 或 `/bl add`、`/bl remove` 管理授权玩家。
- 聊天栏授权支持一次输入多个名字，可用空格或逗号分隔。
- 已授权但非锁主的玩家可查看只读摘要 GUI；未授权玩家不会看到完整授权名单。
- `/bl info` 与 GUI 使用同一套锁摘要语义，显示 owner、权限范围、目标摘要和扩展牌数量。
- 当主锁牌空间不足时，可自动扩展 `[更多用户]` 授权牌。
- 对未授权访问、破坏、自动化搬运、活塞推动、液体流动、爆炸破坏等路径提供保护。
- 兼容传统 `AsyncPlayerChatEvent` 与 Paper 现代 `AsyncChatEvent`，聊天输入授权在新旧聊天事件路径下都能工作。

适合哪些服务器

- 想要轻量容器保护，又不想上来就引入领地系统的生存服。
- 希望保留传统锁牌体验，但想给玩家一个更清晰的 GUI 管理入口。
- 需要兼容 Paper 或 Folia，并尽量减少旧式锁牌插件常见边界问题的服务器。


创建锁

玩家手持普通牌子，对可上锁方块右键即可直接创建锁。  
默认支持容器类方块，也支持门、活板门和栅栏门等交互目标。

 管理授权

- 锁主人普通右键锁牌：打开 GUI 管理界面。
- 点击“添加授权”按钮后，在聊天栏输入玩家名。
- 可一次输入多个名字，支持空格或逗号分隔。
- 输入 `cancel` 可取消本次聊天输入。
- 也可以继续使用命令 `/bl add` 与 `/bl remove`。

查看信息

- 锁主人：可进入完整管理 GUI。
- 已授权玩家：可进入只读摘要 GUI。
- 未授权玩家：无法查看完整锁名单。
- 使用 `/bl info` 时，会输出与 GUI 一致的锁摘要信息。

命令说明

> `add` / `remove` / `info` 默认要求玩家准星指向一个有效的锁牌。

| 命令 | 别名 | 说明 |
| --- | --- | --- |
| `/signlock reload` | 无 | 重载 `config.yml` 与 `gui.yml` |
| `/bl add <玩家名...>` | `/signlock add <玩家名...>` | 添加一个或多个授权玩家 |
| `/bl remove <玩家名...>` | `/signlock remove <玩家名...>` | 移除一个或多个授权玩家 |
| `/bl info` | `/signlock info` | 查看当前锁的摘要信息 |

权限节点

| 权限 | 默认值 | 说明 |
| --- | --- | --- |
| `signlock.add` | `true` | 允许使用授权管理命令 |
| `signlock.reload` | `op` | 允许重载插件配置 |
| `signlock.admin` | `op` | 当 `protection.admin-bypass` 开启时允许管理员绕过锁限制 |

主要配置项


- `signs.lock-header`：主锁牌标题，建议不要随意修改旧服线上值。
- `signs.more-users-header`：扩展授权牌标题。
- `protection.admin-bypass`：是否允许管理员绕过。
- `protection.explosions`：是否保护锁结构不被爆炸破坏。
- `protection.max-more-user-signs`：每把锁最多允许多少块 `[更多用户]` 扩展牌。
- `protection.lockable-materials`：允许上锁的材料白名单。


- `gui.layout`：GUI 布局。
- `gui.legend`：布局字符映射。
- `gui.items.*`：各个 GUI 组件的材质、名称和 Lore。

如果你希望把管理界面做得更符合自己服务器风格，优先调整 `gui.yml` 即可。
