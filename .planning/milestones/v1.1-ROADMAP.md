# Milestone v1.1: Modern UX

**Status:** SHIPPED 2026-04-03  
**Phases:** 4-6  
**Total Plans:** 9

## Overview

v1.1 在不破坏 v1.0 牌子锁语义的前提下，把玩家和服主最常接触的管理体验做成了真正“现代可用”的版本：主人有稳定 GUI，批量授权能一次处理多人，锁状态展示也终于能把“谁能看、谁能管、锁的是哪一个目标”说清楚。

## Phases

### Phase 4: GUI Lock Management

**Goal**: 给锁主人提供稳定、直观的 GUI 管理入口和锁摘要面板，同时保持 v1.0 的牌子语义、命令语义和 shared-target 保护规则不回退。  
**Depends on**: v1.0 release baseline  
**Plans**: 3 plans

Plans:

- [x] 04-01: foundation / test scaffolding / GUI summary model
- [x] 04-02: entry integration / chest GUI rendering / click safety
- [x] 04-03: single add/remove actions / chat input bridge / regression closeout

**Details:**

- 建立了 GUI 会话与摘要模型，让锁主人普通右键锁牌即可进入管理界面。
- 保留潜行右键的原生牌子编辑回退，不打断已有服主习惯。
- GUI 内可以查看 owner、授权玩家、扩展牌数量与当前锁目标，并完成单人 add/remove。

### Phase 5: Batch Authorization Flows

**Goal**: 把授权管理从“一次一人”升级为真正高效的批量操作。  
**Depends on**: Phase 4  
**Plans**: 3 plans

Plans:

- [x] 05-01: shared batch parser / orchestration baseline
- [x] 05-02: single-page GUI batch add/remove flow
- [x] 05-03: batch command compatibility / regression closeout

**Details:**

- GUI 和命令路径现在都支持一次处理多个玩家。
- 批量移除能区分“已移除 / 未找到 / 不可移除”，不会把主人或受保护结果混在一起。
- `/bl add`、`/bl remove` 保留单目标旧写法，老用法没有断。

### Phase 6: Visual Lock Insights & Safety

**Goal**: 让锁状态更容易理解，并把 v1.1 新交互路径的回归与安全覆盖补完整。  
**Depends on**: Phase 5  
**Plans**: 3 plans

Plans:

- [x] 06-01: shared summary contract / human-readable target summary
- [x] 06-02: viewer-aware GUI routing / read-only summary surface
- [x] 06-03: `/bl info` parity / regression and safety closeout

**Details:**

- GUI 与 `/bl info` 现在共用同一套锁摘要语义。
- 已授权但非主人的玩家可以进入只读摘要界面，不再只能靠猜。
- 双箱、单箱和 managed sign 的目标说明统一走 canonical shared-target 语义。
- 配置文案、手测清单和回归测试一起补齐，确保现代化体验没有换来安全回退。

## Milestone Summary

### Key Decisions

- 保持牌子数据结构不变，只在表现层增加 GUI、批量操作与摘要语义。
- owner 普通右键进 GUI，潜行右键仍保留原生牌子编辑，兼顾现代体验和服主兼容性。
- `/bl` 命令持续保留，并与 GUI 共享同一套授权与摘要逻辑，而不是重新发明第二套规则。
- 把“可访问”和“可管理”显式展示出来，避免玩家误判权限边界。

### Issues Resolved

- 锁管理不再只能依赖命令和手改牌子。
- 授权名单大的时候，不再需要重复执行大量单人命令。
- 双箱和共享目标的锁摘要终于能稳定表达，不再因入口不同而漂移。
- v1.1 新增 GUI 路径的保护回归和手测基线都被补齐。

### Issues Deferred

- 审计面板与更完整的管理后台
- 搜索、分页、筛选式授权管理
- 更深层的运营能力，例如日志追踪与跨服同步

### Technical Debt

- milestone audit 没有在 closeout 前单独执行，这次归档基于各 phase 的 verification 报告完成。
- `plugin.yml`、`.planning/config.json` 和部分早期 phase 文件仍带着工作区里的独立改动，未纳入本次里程碑归档提交。
