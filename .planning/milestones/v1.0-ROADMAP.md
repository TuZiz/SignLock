# Milestone v1.0: Initial Release

**Status:** SHIPPED 2026-04-02  
**Phases:** 1-3  
**Total Plans:** 9

## Overview

v1.0 将 SignLock 从“已有核心代码但行为不稳定”的 brownfield 项目，收口成可发布的中文服牌子锁插件。这个里程碑优先解决锁目标一致性、授权边界、自动化防绕过、中文默认资源和发布交付，而不是扩展 GUI 或跨服能力。

## Phases

### Phase 1: Lock Target Correctness

**Goal**: 让主锁牌、扩展牌、单箱、双箱和容器访问在所有入口上都解析到同一受保护目标。  
**Depends on**: Nothing (first phase)  
**Plans**: 3 plans

Plans:

- [x] 01-01: 安装 MockBukkit 回归基础并统一 `LockService` 的 canonical target 语义
- [x] 01-02: 让 `LockListener` 在交互、开箱、自动化入口复用 shared-target 结论
- [x] 01-03: 收口锁牌放置与锁牌编辑保护

### Phase 2: Authorization & Protection Matrix

**Goal**: 覆盖授权管理、破坏保护、自动化保护和身份解析的完整保护矩阵。  
**Depends on**: Phase 1  
**Plans**: 3 plans

Plans:

- [x] 02-01: 先补 listener 保护、命令授权和身份缓存的回归测试
- [x] 02-02: 修正 `LockService` / `LockListener` 的破坏权限和环境保护矩阵
- [x] 02-03: 收口命令身份归一化并重跑完整回归

### Phase 3: Localization & Release Readiness

**Goal**: 让默认资源、插件元数据与发布验证达到交付标准。  
**Depends on**: Phase 2  
**Plans**: 3 plans

Plans:

- [x] 03-01: 现代化运行时默认文案与运维反馈
- [x] 03-02: 产出快速开始、手测矩阵与发布前检查
- [x] 03-03: 运行最终构建并生成发布验证报告

## Milestone Summary

### Key Decisions

- 使用 brownfield 稳定化路线，而不是重写插件
- 双箱两半必须视为同一把锁的 shared target
- 授权矩阵以“可访问 / 可管理 / 可破坏”三层语义收口
- “更现代化”在 v1.0 只落到文案、配置可读性和交付体验，不扩展成新系统

### Issues Resolved

- 修复了双箱/共享容器锁目标漂移导致的保护失效
- 修复了授权玩家、管理员与普通玩家边界不清的问题
- 修复了自动化、活塞、水流、爆炸等绕过路径
- 修复了默认中文资源与发布文档不可直接交付的问题

### Issues Deferred

- GUI 化锁管理
- 批量授权 / 批量移除
- 审计面板与更完整的现代交互流程
- 里程碑级 audit 仍可作为后续补充动作

### Technical Debt

- `plugin.yml` 仍带有用户工作区中的本地改动，本次归档按当前工作树状态记录，但未额外改写
- Phase 执行历史保留在 `.planning/phases/`，后续如需搬迁可再用 `$gsd-cleanup`
