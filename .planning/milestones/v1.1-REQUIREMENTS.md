# Requirements Archive: v1.1 Modern UX

**Archived:** 2026-04-03  
**Status:** SHIPPED  
**Result:** 12 / 12 v1.1 requirements completed

## Validated Requirements

### GUI Management

- [x] `GUI-01` 锁主人可以通过稳定入口打开锁管理界面，而不必只依赖命令和手改牌子
- [x] `GUI-02` 锁管理界面能显示所有者、授权玩家、扩展牌数量和当前锁目标摘要
- [x] `GUI-03` 锁主人可以在 GUI 中执行单人授权与移除，而不破坏原有牌子数据格式

### Batch Authorization

- [x] `BATCH-01` 锁主人可以一次添加多个玩家到同一把锁
- [x] `BATCH-02` 锁主人可以一次移除多个玩家，且结果能区分成功、未找到和不可移除
- [x] `BATCH-03` 批量授权能力在 GUI 与命令路径上保持一致语义，不能绕过所有权检查

### Visual Lock Info

- [x] `VIS-01` 玩家可以更直观地查看锁的所有者、授权列表和扩展牌使用情况
- [x] `VIS-02` GUI 与信息展示明确区分“可访问”和“可管理”边界
- [x] `VIS-03` 锁信息展示兼容单箱、双箱与 managed sign 的 shared-target 语义

### Compatibility & Safety

- [x] `SAFE-01` v1.0 的牌子创建、扩展牌放置、命令授权和保护矩阵行为没有回退
- [x] `SAFE-02` 新交互没有破坏 Spigot / Paper / Folia 兼容声明和主线程安全边界
- [x] `SAFE-03` 仓库补齐了对应的自动化测试与手测文档，覆盖 GUI / 批量授权 / 摘要主路径
