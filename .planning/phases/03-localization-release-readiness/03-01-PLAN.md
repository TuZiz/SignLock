# Plan 03-01: Repair Runtime Resources and Operator Feedback

**Phase:** 3 - Localization & Release Readiness
**Wave:** 1
**Status:** Planned

## Objective

Make the shipped runtime-facing experience readable and modern for Chinese operators without changing plugin behavior.

## Scope

- repair garbled Chinese in `plugin.yml`
- repair and modernize readable Chinese defaults in `config.yml`
- align reload / info / startup-facing feedback with the agreed concise style
- keep config keys and runtime semantics compatible

## Primary Files

- `src/main/resources/plugin.yml`
- `src/main/resources/config.yml`
- `src/main/java/ym/signLock/SignLock.java`
- `src/main/java/ym/signLock/config/SignLockConfig.java`

## Tasks

1. Audit all operator-facing metadata and message defaults currently shipped from resources.
2. Rewrite garbled or outdated Chinese into concise, modern Chinese while preserving message meaning.
3. Verify runtime code still reads the same keys and message paths after the cleanup.
4. Add or adjust reload / startup feedback only where existing runtime seams already exist.
5. Build or smoke-check resource loading so the cleanup does not introduce missing-key regressions.

## Must Hold True

- no config key renames
- no permission or command registration regressions
- no change to lock/protection semantics
- default Chinese is directly readable in UTF-8

## Verification Target

- readable `plugin.yml`
- readable `config.yml`
- working reload/info/startup feedback path
